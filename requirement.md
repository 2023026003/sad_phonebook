# 전화번호 관리 시스템 요구사항

## 프로젝트 개요

웹 기반의 전화번호부(Phonebook) 관리 시스템으로, 연락처 정보를 손쉽게 등록·수정·삭제할 수 있는 CRUD 애플리케이션입니다.

---

## 기술 스택

| 구분 | 기술 |
|------|------|
| 프론트엔드 | HTML, CSS, JavaScript (또는 Next.js) |
| 백엔드/DB | [Supabase](https://supabase.com) (PostgreSQL 기반) |
| 배포/호스팅 | [Vercel](https://vercel.com) |

---

## 기능 요구사항

### 1. 전화번호 등록 (Create)
- 이름과 전화번호를 입력하여 새 연락처를 추가할 수 있다.
- 필수 입력값: 이름, 전화번호
- 선택 입력값: 이메일, 메모 등 (추가 구현 시)

### 2. 전화번호 조회 (Read)
- 등록된 전체 연락처 목록을 화면에 표시한다.
- 이름 또는 전화번호로 검색할 수 있다. (선택 사항)

### 3. 전화번호 수정 (Update)
- 기존 연락처의 이름 및 전화번호를 수정할 수 있다.
- 수정 후 목록에 즉시 반영된다.

### 4. 전화번호 삭제 (Delete)
- 선택한 연락처를 목록에서 삭제할 수 있다.
- 삭제 전 확인 절차(confirm)를 거친다.

---

## 비기능 요구사항

- **인증 없음**: 로그인 및 회원가입 기능은 구현하지 않는다.
- **단순성**: 기본 CRUD 기능만 구현하며, 복잡한 비즈니스 로직은 포함하지 않는다.
- **반응형 UI**: 모바일 및 데스크톱 환경에서 정상적으로 동작해야 한다.

## 데이터베이스 설계 (Supabase)

### 테이블명: `contacts`

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | `uuid` (PK) | 자동 생성되는 고유 식별자 |
| `name` | `text` | 연락처 이름 (필수) |
| `phone` | `text` | 전화번호 (필수) |
| `created_at` | `timestamptz` | 생성일시 (자동) |
| `updated_at` | `timestamptz` | 수정일시 (자동) |

---

## 배포 환경

- **Supabase**: 데이터베이스 및 REST API 제공
  - Supabase 프로젝트 URL 및 `anon` 공개 키를 환경 변수로 관리
- **Vercel**: 정적 사이트 또는 Next.js 앱 배포
  - 환경 변수: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
